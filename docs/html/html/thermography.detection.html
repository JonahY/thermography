clean_angle_consistency(max_neighbor_angle: float)¶

Removes the segments whose angle differs more than the passed threshold to the mean angle of its two neighbors.

Parameters:max_neighbor_angle – Maximal angle allowed between each segments and its two neighbors. If their angle differs more than this parameter, the central segment is discarded.
clean_clusters(mean_angles: numpy.ndarray, max_angle_variation_mean: float = 0.3490658503988659, max_merging_angle: float = 0.08726646259971647, max_endpoint_distance: float = 50, max_neighbor_angle=0.05235987755982989)¶

Cleans the clusters by removing edges outliers (angle deviation from cluster mean is too high), and by merging almost collinear segments into a single segment.

Parameters:
  • mean_angles – List of mean angles computed for each cluster.
  • max_angle_variation_mean – Maximal allowed angle between each segment and corresponding cluster mean angle.
  • max_merging_angle – Maximal allowed angle between two segments in order to merge them into a single one.
  • max_endpoint_distance – Maximal summed distance between segments endpoints and fitted line for merging segments.
  • max_neighbor_angle – Maximal angle allowed between each segments and its two neighbors. If their angle differs more than this parameter, the central segment is discarded.
clean_clusters_angle(mean_angles: numpy.ndarray, max_angle_variation_mean: float)¶

Removes all segments whose angle deviates more than the passed parameter from the mean cluster angle.

Parameters:
  • mean_angles – List of cluster means.
  • max_angle_variation_mean – Maximal angle variation to allow between the cluster segments and the associated mean angle.
cluster_segments(num_clusters: int = 15, n_init: int = 10, cluster_type: str = 'gmm', swipe_clusters: bool = True, use_angles: bool = True, use_centers: bool = False)¶

Clusters the input segments based on the parameters passed as argument. The features that can be used to cluster the segments are their mean coordinates, and their angle.

Parameters:
  • num_clusters – Number of clusters to extract from the parameter space.
  • n_init – Number of initializations to be performed when clustering.
  • cluster_type – Clustering algorithm to be used, must be in [‘gmm’, ‘knn’] which correspond to a full gaussian mixture model, and k-nearest-neighbors respectively.
  • swipe_clusters – Boolean flag, if set to ‘True’ and ‘cluster_type’ is ‘gmm’, then the algorithm iterates the clustering procedure over a range of number of clusters from 1 to ‘num_clusters’ and retains the best result.
  • use_angles – Boolean flag indicating whether to consider angles in the clustering process.
  • use_centers – Boolean flag indicating whether to consider segment centroids in the clustering process.
compute_cluster_mean() → tuple¶

Computes the mean values (coordinates and angles) for each one of the identified clusters.

Returns:The mean angles, and mean coordinates of each cluster.
merge_collinear_segments(max_merging_angle: float, max_endpoint_distance: float)¶

Merges all collinear segments belonging to the same cluster.

Parameters:
  • max_merging_angle – Maximal angle to allow between segments to be merged.
  • max_endpoint_distance – Maximal summed distance between segments endpoints and fitted line for merging segments.
plot_segment_features()¶

Plots the first two dimensions of the features used for clustering.
